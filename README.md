# MedMemo · 医学知识理解性记忆系统

> 为「**理解性记忆 + 持久性记忆**」而设计，不是普通闪卡工具。

## 为什么不一样

普通闪卡只做"正反翻面"，本质是再认，忘得快。MedMemo 落地了 6 条有认知科学支撑的机制：

| 机制 | 作用 | 怎么实现的 |
|------|------|-----------|
| **提取练习** | 持久记忆 | 答案默认隐藏，逼你先在脑中回忆再看 |
| **SM-2 间隔重复** | 持久记忆 | Anki 同款算法，会了的往后排，忘的明天重来 |
| **合意困难（交错）** | 持久记忆 | 复习队列按知识点/学科打散，不连续练同类 |
| **多角度提问** | 理解记忆 | 一个知识点生成 正向/反向/机制/应用 四种问法 |
| **对比辨析** | 理解记忆 | AI 自动识别易混概念，生成对照卡（如 1型 vs 2型糖尿病）|
| **精细加工** | 理解+持久 | 每个知识点附「机制因果链 + 临床联系 + 记忆画面」|
| **大小分级** | 理解记忆 | AI 自动区分大小知识点，小事实只出 1 卡，大概念才出 4 卡 |

## 大小知识点

不是所有知识点都需要 4 种问法。一句话能说清的孤立事实（某正常值、某限速酶、某结构位置）用**小知识点**最合适：

| | 大知识点 `big` | 小知识点 `small` |
|---|---|---|
| 适用 | 需要理解因果链的概念（药机制、病病理） | 只需死记一个结论的事实 |
| 卡片 | 正向/反向/机制/应用 4 张 | 仅 1 张正向卡 |
| 理解层 | 机制 + 临床 + 记忆画面 + 层级节点 | 全部留空 |
| 知识网络 | 学科色圆点（大） | 灰色菱形（小），一眼区分 |

**AI 自动判别**：粘贴文本时，AI 会逐条判断大小并自动生成；同时比对已有知识点，疑似重复的会标红提示（不自动跳过，由你决定是否入库）。

## 快速开始

### 1. 启动（零依赖）
双击 `start.bat`，或命令行运行：
```
python server.py
```
浏览器自动打开 `http://localhost:8000`。

> 首次启动会自动导入 6 个医学多学科示例知识点（药理/病理/内科/生理/解剖），开箱即用。

### 2. 启用 AI 智能拆卡（可选）
打开 `config.json`，填入你的 DeepSeek API key：
```json
{
  "deepseek_api_key": "sk-你的key",
  "deepseek_model": "deepseek-chat",
  "new_cards_per_day": 0
}
```
重启服务器后，到「AI 导入」页粘贴教材文本，AI 会自动拆卡。
拆卡结果会先进入审核区，入库前可以直接修改知识点字段、卡片问题/答案，或删除不想要的知识点、对照卡。审核区会自动标出空标题、缺卡片、重复标题、疑似重复已有知识点、答案过长等质量问题。
如果 AI 返回的 JSON 被截断、混入多余文字或有轻微格式错误，后端会先尝试自动修复；修不完整时也会尽量保留已经解析完整的知识点，并在审核区提示你重点检查。

> 没有 key 也能用：手动在主页「新增知识点」即可，AI 功能只是锦上添花。
> 获取 key：https://platform.deepseek.com/

主页「我的知识点」支持按标题、理解字段和卡片内容即时搜索；配合学科筛选，可以快速找回某个概念、病例表现或问题答案。

已有知识点可以在主页直接编辑标题、学科、理解字段和卡片；保存时后端会用单个事务更新，避免只改了一半。

知识点列表支持多选后批量改学科、批量删除，也可以导出选中的知识点或当前学科的独立备份，适合知识点变多后的整理和迁移。

## 使用流程

```
粘贴医学教材/笔记
        │
        ▼
  AI 拆卡（选学科 → 自动套模板）
   ┌────────────────────────────┐
   │ • 原子化拆分（一卡一问）    │
   │ • 生成机制/临床/记忆画面    │
   │ • 4 种问法卡（正反机应）    │
   │ • 易混概念对照卡            │
   └────────────────────────────┘
        │  你审核
        ▼
      入库
        │
        ▼
   每日复习（SM-2 调度 + 交错出题）
```

## 复习交互

- **点击卡片 / 按空格**：翻面看答案（先想！别急着翻）
- **数字键 1/2/3/4** 或点击按钮评分：
  - `1` 重记（完全忘了 → 明天再来）
  - `2` 困难（勉强想起）
  - `3` 良好（正常答出 → 正常推进间隔）
  - `4` 简单（秒答 → 加速推进）
- **错题重练**：`重记/困难` 会进入或留在错题池；在错题重练中连续 2 次 `良好/简单` 后自动毕业。以后如果再次答错，会重新进入错题池。
- **错题分析**：首页会把当前错题池按知识点聚合，显示最需要优先回看的薄弱点；点击薄弱点可直接定位到对应知识点。
- **撤销评分**：手滑评错时，可以撤销上一张卡的评分；卡片间隔、复习历史和错题池状态都会一起回滚。
- **本轮进度**：复习页会显示剩余卡片、完成度、本轮正确率和预计剩余时间；撤销评分时，这些本轮统计也会同步回滚。
- **结束总结**：一轮复习结束后，会汇总本轮题数、正确率、用时、需要优先回看的知识点和本轮错题。

## 知识网络

「知识网络」页把知识点从线性卡片变成可探索的图谱，适合复盘一个学科里“概念之间怎么互相牵动”。

### 1. 知识点网络

- **根节点**：每个知识点都会生成一个根节点，颜色按学科区分。
- **层级展开**：点击大知识点可以展开 AI 生成的子模块和关键词，例如「病因 → 发病机制 → 临床表现 → 治疗」。
- **小知识点归属**：小知识点默认不会散成一片；可以点击「关联小知识点」，让 AI 把它们归属到同学科最相关的大知识点下。
- **跨知识点虚线**：如果某个子节点语义上对应另一个独立知识点（如「地高辛」对应「地高辛的作用机制」），网络会用虚线把两者连起来。
- **详情面板**：点击节点可以查看知识点说明、卡片、机制/临床/记忆画面等信息。
- **从网络复习**：在节点详情里点击「复习此知识点」，会直接进入该知识点的定向复习，包含该点的全部卡片。

### 2. 对比网络

AI 导入识别到易混概念时，会生成结构化对比维度；「对比网络」会把这些概念连成图。

- 同一个概念可以参与多组对比，自动汇聚成网络。
- 边上的标签显示对比维度，例如「病因 / 表现 / 治疗」。
- 点击概念或连线，可以查看完整对比表，适合考前快速辨析。

### 3. 编辑模式

网络页提供轻量编辑能力：

- 手动新增子节点，补充 AI 没拆出来的结构。
- 编辑节点名称和说明。
- 删除不需要的节点。
- 添加自定义连线，把自己认为有关联的节点连起来。

这些结构都会写入本地 SQLite 数据库，并随「导出备份」一起保存。

## 学科模板（AI 自动套用）

| 学科 | 拆解维度 |
|------|---------|
| 药理 | 机制 → 作用 → 用途 → 不良反应 → 禁忌 |
| 病理 | 病因 → 病变 → 临床病理联系 |
| 生理 | 概念 → 机制 → 调节 → 失衡后果 |
| 解剖 | 位置 → 结构 → 毗邻 → 功能 |
| 内科 | 表现 → 诊断 → 鉴别 → 治疗 |
| 外科 | 表现 → 诊断 → 手术 → 并发症 |

## 数据备份

主页统计条右侧有 **⬇ 导出备份**，点击即下载一份完整备份（含知识点、卡片、SM-2 状态、关联、知识网络、对比网络、手动连线、复习历史）。文件名带日期，可多次导出留存多个版本。

旁边的 **↥ 恢复备份** 可以选择之前导出的 JSON 文件恢复。恢复前会先显示备份摘要（导出时间、知识点数、卡片数、复习记录和网络数据量），并自动把当前数据保存到本机 `backups/` 目录作为安全快照；随后清空当前全部学习数据，再替换为备份内容。后端使用单个 SQLite 事务执行，失败会回滚，不会留下半截数据。

服务每天首次启动时也会在本机 `backups/` 目录自动保存一份 `auto-日期-时间.json`，默认只保留最近 7 份自动备份。

> 单文件 SQLite（`flashcards.db`）一旦损坏或误删就全没了。建议定期导出备份，重要考试前必导一次。
> 备份文件不包含 `config.json` 或 API key。

## 配置项

`config.json` 除 AI key 外，还支持：

| 字段 | 默认 | 说明 |
|------|------|------|
| `deepseek_base_url` | `https://api.deepseek.com` | DeepSeek/OpenAI 兼容接口地址。 |
| `deepseek_model` | `deepseek-chat` | AI 拆卡使用的模型。 |
| `deepseek_max_tokens` | `8192` | AI 单次返回的最大 token 数；长文本拆卡时可适当调大，减少输出被截断的概率。 |
| `host` | `localhost` | 服务监听地址。本机使用保持 `localhost`；需要局域网访问时可改为 `0.0.0.0`。 |
| `port` | `8000` | 服务端口。 |
| `new_cards_per_day` | `0` | 新卡每日上限。`0` = 不限（AI 批量拆卡后全部进入队列）；设为正数（如 `20`）则每天只推这么多新卡，其余排队，防止复习量雪崩。仅对「按科目」复习生效——主动选知识点复习时不截断。 |
| `auto_backup_keep` | `7` | 自动备份保留份数。每天首次启动自动备份一次；只清理 `auto-` 开头的自动备份，不影响手动导出的备份。 |

## 关于记忆机制的原理

- **提取 > 重读**：主动回忆一次比被动看十遍有效。所以答案默认隐藏。
- **间隔 > 集中**：遗忘曲线是指数的，在你"快忘"时复习，记忆最牢。SM-2 自动算时机。
- **多角度 > 单一**：一个知识点能从多个方向提取，才算真懂。所以生成 4 种问法。
- **对比 > 孤立**：易混概念放一起对比，比单记两个省力。所以自动找易混点。
- **画面 > 文字**：大脑对生动场景的记忆碾压文字。所以配"记忆画面"描述，让你脑内成像。
